3  Resolución de grandes sistemas de ecuaciones

Este tema se centra en la resolución de grandes sistemas de ecuaciones compatibles determinados.

3.1 Ejemplo de la barra metálica

Imaginemos una barra metálica de 1 metro, cuyo extremo izquierdo mantenemos a \(0ºC\) y el extremo derecho a \(100ºC\). Queremos determinar la temperatura estacionaria de toda la barra.

Sabiendo que:

\[ T_i = \frac{T_{i+1} + T_{i-1}}{2} \]

donde \(T_i\) es la temperatura en el i-ésimo punto de la barra, podemos obtener un sistema de ecuaciones:

\[ T_{i-1} - 2 T_{i} + T_{i+1} = 0, \space \forall i \]

3.2 R para sistemas de ecuaciones

3.2.1 Creación de matrices en R:

Nuestro objetivo es representar el siguiente sistema de ecuaciones:

\[ \begin{pmatrix} 1 & 2 & 3 \\ 4 & 5 & 6 \\ 1 & 2 & 4 \end{pmatrix} \begin{pmatrix} x \\ y \\ z \end{pmatrix} = \begin{pmatrix} 0 \\ 0 \\ 1 \end{pmatrix} \] cuya notación matricial:

\[ Ax=b \]

A <- matrix(
  c(1, 2, 3,
    4, 5, 6,
    1, 2, 4),
  ncol = 3, byrow = TRUE
)

b <- matrix(
  c(0, 0, 1),
  byrow=TRUE
)

# Método de R base
res <- solve(A, b)

Podemos verificarlo mediante:

A %*% res
     [,1]
[1,]    0
[2,]    0
[3,]    1

Sin embargo, ¿Qué pasa cuando A no es regular o el sistema es incompatible?

A_singular <- matrix(
  c(1, 2, 3,
    4, 5, 6,
    1, 2, 3),
  ncol = 3, byrow = TRUE
)

#solve(A_singular, b)

3.2.2 Factorización \(LU\)

En R, podemos usar el paquete Matrix para obtener la descomposición \(LU\):

library(Matrix)

lu_A <- lu(A)
matrices_A <- expand(lu_A)
matrices_A$L %*% matrices_A$U
3 x 3 Matrix of class "dgeMatrix"
     [,1] [,2] [,3]
[1,]    4    5    6
[2,]    1    2    3
[3,]    1    2    4

Sabemos que \(PA = LU\)

# 1. Construyo el sistema PA = LU
nueva_A <- matrices_A$P %*% A
nueva_b <- matrices_A$P %*% b

# 2. Resuelvo L(Ux) = Pb
# 2.1 Ly = Pb (siendo y = Ux)
y <- forwardsolve(matrices_A$L, nueva_b)

# 2.2 Ux = y
x <- backsolve(matrices_A$U, y)

3.2.3 Factorización de Cholesky

Si la matriz \(A\) es simétrica y definida positiva, existe una única matriz triangular inferior \(L\) tal que:

\[A = LL^T\] En R, podemos usar la función chol:

A <- matrix(
  c(4, 2, -2,
    2, 10, 2,
    -2, 2, 5),
  nrow = 3, byrow = TRUE
)
b <- c(6, 15, 12)

R <- chol(A)
R
     [,1] [,2]      [,3]
[1,]    2    1 -1.000000
[2,]    0    3  1.000000
[3,]    0    0  1.732051

Podemos recuperar \(A\) mediante:

t(R) %*% R
     [,1] [,2] [,3]
[1,]    4    2   -2
[2,]    2   10    2
[3,]   -2    2    5

Primero hay que resolver \(R^Ty=b\)

y <- forwardsolve(t(R), b)

Luego, \(Rx=y\)

x <- backsolve(R, y)

3.2.4 Factorización \(QR\)

\[A = QR\]

donde \(R\) es triangular superior y \(Q\) es ortogonal, es decir, \(Q^TQ=I\). Por tanto si se nos presenta el siguiente sistema:

\[ Ax=b \]

\[ QRx=b \\ \]

\[ Q^TQRx=Q^Tb \implies Rx=Q^Tb \]

En R, podemos usar la función qr.solve():

qr.solve(A, b)
[1] 3.2777778 0.1111111 3.6666667

Si hacemos qr.solve(A_singular, b) se da cuenta y da error.

También podemos obtener las matrices:

decomp_qr <- qr(A)
Q <- qr.Q(decomp_qr)
Q

3.3 Trabajar con matrices dispersas

Almacenar un 0 ocupa lo mismo que almacenar \(\pi\) o cualquier otro número. En matrices dispersas, esto es un problema.

\[ \text{sparsity} = 1 - \frac{\text{número de elementos no nulos(NNZ)}}{\text{número total de elementos}(n \times m)} \]

¿Cómo podemos almacenar dichas matrices?

Supongamos la siguiente matriz:

\[ A = \begin{pmatrix} 1 & 0 & 0 \\ 1 & 2 & 0 \\ 0 & 0 & 3 \\ \end{pmatrix} \]

A <- matrix(
  c(1, 0, 0,
    0, 2, 0,
    0, 0, 3),
  ncol = 3, byrow = TRUE
)

Una forma podría ser:

# Este método es el default en el paquete Matrix
i <- c(1, 2, 2, 3)
p <- c(0, 2, 3, 4)
x <- c(1, 1, 2, 3)

Otra posibilidad:

# Este método nos interesa más por motivos de sencillez
i <- c(1, 2, 2, 3)
j <- c(1, 1, 2, 3)
x <- c(1, 0, 2, 3)

3.3.1 Matrices banda y tridiagonales

Ejemplo de clase: crear una matriz \(A\) 10000x10000 que tenga en la diagonal principal un 3, justo por encima un -1 y justo por debajo -1. Además, crear un vector \(b\) 10000x1 cuyo último elemento es \(\pi\):

n <- 10000

# Definir las diagonales
diag_principal <- rep(3, n)
sub_diag <- rep(-1, n-1)
super_diag <- rep(-1, n-1)

# Definir las posiciones (i, j)
i <- c(1:n, 2:n, 1:(n-1))
j <- c(1:n, 1:(n-1), 2:n)
x <- c(diag_principal, sub_diag, super_diag)

# Usamos sparseMatrix
A <- sparseMatrix(i = i, j = j, x = x)

3.4 Métodos Iterativos para sistemas grandes

3.4.0.1 Método de Jacobi

Supongamos el siguiente sistema de ecuaciones:

\[ \begin{cases} x + 2y = 3 \\ 3x - y = 4 \end{cases} \] Despejamos la variable i-ésima en la i-ésima ecuación:

\[ \begin{cases} x = 3 - 2y \\ y = 3x - 4 \end{cases} \]

Que expresado de forma matricial:

$$

\[\begin{pmatrix} x \\ y \end{pmatrix}\] = \[\begin{pmatrix} 0 & -2 \\ 3 & 0 \end{pmatrix} \begin{pmatrix} x \\ y \end{pmatrix}\]
  • \[\begin{pmatrix} 3 \\ -4 \end{pmatrix}\] $$

\[ \vec{x}^{(k+1)} = M \vec{x}^{(k)} + C \]

Por tanto:

\[ x^{(k+1)} = 3 - 2y^{(k)} \\ y^{(k+1)} = 3x^{(k)} - 4 \]

En R podemos usar la librería Rlinsolve:

library(Rlinsolve)
